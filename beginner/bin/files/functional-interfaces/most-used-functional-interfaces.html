Em Java, as interfaces funcionais são interfaces que possuem apenas um método abstrato (SAM - Single Abstract Method) e são amplamente utilizadas em expressões lambda e programação funcional. A partir do Java 8, muitas interfaces funcionais foram introduzidas no pacote java.util.function, além de outras já existentes em versões anteriores.

Aqui estão as interfaces funcionais mais usadas em Java:

🔹 Principais interfaces do pacote java.util.function

  1. Predicate<T>

    • Método abstrato: boolean test(T t)
    • Uso: Verifica uma condição e retorna true ou false.
    • Exemplo:
      Predicate<Integer> isEven = num -> num % 2 == 0;
System.out.println(isEven.test(10)); // true

  2. Function<T, R>

    • Método abstrato: R apply(T t)
    • Uso: Transforma um valor de um tipo (T) em outro (R).
    • Exemplo:
      Function<String, Integer> strToLength = s -> s.length();
System.out.println(strToLength.apply("Java")); // 4

  3. Consumer<T>

    • Método abstrato: void accept(T t)
    • Uso: Executa uma ação com um argumento, mas não retorna nada.
    • Exemplo:
      Consumer<String> printUpperCase = s -> System.out.println(s.toUpperCase());
printUpperCase.accept("hello"); // HELLO

  4. Supplier<T>

    • Método abstrato: T get()
    • Uso: Fornece um valor sem receber argumentos (como um “fabricante”).
    • Exemplo:
      Supplier<Double> randomValue = () -> Math.random();
System.out.println(randomValue.get()); // 0.123...

  5. UnaryOperator<T> (herda Function<T, T>)

    • Método abstrato: T apply(T t)
    • Uso: Operação que recebe e retorna o mesmo tipo.
    • Exemplo:
      UnaryOperator<Integer> square = x -> x * x;
System.out.println(square.apply(5)); // 25

  6. BinaryOperator<T> (herda BiFunction<T, T, T>)

    • Método abstrato: T apply(T t1, T t2)
    • Uso: Combina dois valores do mesmo tipo e retorna um resultado do mesmo tipo.
    • Exemplo:
      BinaryOperator<Integer> sum = (a, b) -> a + b;
System.out.println(sum.apply(3, 7)); // 10

🔹 Outras interfaces funcionais úteis

  1. BiFunction<T, U, R>

    • Método: R apply(T t, U u)
    • Uso: Recebe dois argumentos (T e U) e retorna um resultado (R).
    • Exemplo:
      BiFunction<Integer, Integer, String> concatNumbers = (a, b) -> a + "-" + b;
System.out.println(concatNumbers.apply(10, 20)); // "10-20"

  2. BiPredicate<T, U>

    • Método: boolean test(T t, U u)
    • Uso: Avalia dois argumentos e retorna um boolean.
    • Exemplo:
      BiPredicate<String, Integer> checkLength = (s, len) -> s.length() == len;
System.out.println(checkLength.test("Java", 4)); // true

  3. Runnable (antes do Java 8)

    • Método: void run()
    • Uso: Executa uma tarefa sem argumentos ou retorno.
    • Exemplo:
      Runnable task = () -> System.out.println("Running...");
new Thread(task).start();

  4. Comparator<T> (antes do Java 8)

    • Método: int compare(T o1, T o2)
    • Uso: Compara dois objetos para ordenação.
    • Exemplo:
      Comparator<String> byLength = (s1, s2) -> s1.length() - s2.length();
List<String> words = Arrays.asList("Java", "Python", "C");
words.sort(byLength); // [C, Java, Python]

🔹 Resumo das interfaces mais usadas

Interface Método Descrição
Predicate<T> boolean test(T t) Testa uma condição (true/false).
Function<T, R> R apply(T t) Transforma T em R.
Consumer<T> void accept(T t) Consome um valor (sem retorno).
Supplier<T> T get() Fornece um valor (sem entrada).
UnaryOperator<T> T apply(T t) Operação unária (mesmo tipo).
BinaryOperator<T> T apply(T t1, T t2) Operação binária (mesmo tipo).

📌 Observações importantes:

  • Todas essas interfaces podem ser usadas com expressões lambda ou method references (::).
  • Muitas delas possuem métodos default úteis, como and(), or(), negate() (no Predicate), compose() (no Function), etc.
  • Algumas interfaces, como Runnable e Comparator, existiam antes do Java 8, mas se tornaram funcionais com a introdução de lambdas.

Essas interfaces são essenciais para programação funcional em Java, especialmente quando trabalhamos com Streams API e processamento de coleções. 🚀
