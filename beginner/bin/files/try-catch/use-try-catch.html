Em Java, o try-catch é usado para tratar exceções (erros que ocorrem durante a execução de um programa). Ele permite que você tente executar um bloco de código (try) e capture exceções (catch) que possam surgir, evitando que o programa quebre inesperadamente.

🔹 Sintaxe básica do try-catch

try {
    // Código que pode lançar uma exceção
} catch (TipoDaExceção e) {
    // Bloco que trata a exceção
}

Exemplo 1: Tratando uma exceção simples (ArithmeticException)

public class Exemplo1 {
    public static void main(String[] args) {
        try {
            int resultado = 10 / 0; // Lança ArithmeticException (divisão por zero)
            System.out.println("Resultado: " + resultado);
        } catch (ArithmeticException e) {
            System.out.println("Erro: Divisão por zero não permitida!");
        }
    }
}

Saída:

Erro: Divisão por zero não permitida!

🔹 Tratando múltiplas exceções

Você pode capturar diferentes tipos de exceções em blocos catch separados.

Exemplo 2: ArrayIndexOutOfBoundsException e NumberFormatException

public class Exemplo2 {
    public static void main(String[] args) {
        try {
            int[] numeros = {1, 2, 3};
            System.out.println(numeros[5]); // Lança ArrayIndexOutOfBoundsException
            
            String texto = "abc";
            int numero = Integer.parseInt(texto); // Lança NumberFormatException
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("Índice do array inválido!");
        } catch (NumberFormatException e) {
            System.out.println("Não foi possível converter para número!");
        }
    }
}

Saída:

Índice do array inválido!

🔹 Bloco finally

O bloco finally sempre é executado, independentemente de ocorrer uma exceção ou não. É usado para liberar recursos (como fechar arquivos ou conexões de banco de dados).

Exemplo 3: finally em ação

import java.io.FileWriter;
import java.io.IOException;

public class ExemploFinally {
    public static void main(String[] args) {
        FileWriter arquivo = null;
        try {
            arquivo = new FileWriter("dados.txt");
            arquivo.write("Olá, mundo!");
        } catch (IOException e) {
            System.out.println("Erro ao escrever no arquivo: " + e.getMessage());
        } finally {
            try {
                if (arquivo != null) {
                    arquivo.close(); // Garante que o arquivo será fechado
                }
            } catch (IOException e) {
                System.out.println("Erro ao fechar o arquivo: " + e.getMessage());
            }
        }
    }
}

🔹 Capturando exceções genéricas (Exception)

Se você não sabe qual exceção pode ocorrer, pode capturar a superclasse Exception.

Exemplo 4: Tratamento genérico

public class Exemplo4 {
    public static void main(String[] args) {
        try {
            String texto = null;
            System.out.println(texto.length()); // NullPointerException
        } catch (Exception e) {
            System.out.println("Ocorreu um erro: " + e.getMessage());
        }
    }
}

Saída:

Ocorreu um erro: Cannot invoke "String.length()" because "texto" is null

🔹 Lançando exceções manualmente (throw)

Você pode forçar o lançamento de uma exceção com throw.

Exemplo 5: throw em uma validação

public class Exemplo5 {
    public static void main(String[] args) {
        try {
            validarIdade(15);
        } catch (IllegalArgumentException e) {
            System.out.println(e.getMessage());
        }
    }

    static void validarIdade(int idade) {
        if (idade < 18) {
            throw new IllegalArgumentException("Menor de idade não permitido!");
        }
        System.out.println("Acesso liberado.");
    }
}

Saída:

Menor de idade não permitido!

📌 Resumo das estruturas de tratamento de exceções

Estrutura Descrição
try Bloco onde o código que pode lançar exceções é executado.
catch Captura e trata exceções específicas.
finally Sempre executado (usado para liberar recursos).
throw Lança uma exceção manualmente.
throws Declara que um método pode lançar exceções (usado em assinaturas).

✅ Boas práticas

  1. Seja específico: Capture exceções específicas (ArithmeticException, NullPointerException) em vez de Exception genérica.
  2. Use finally para recursos: Sempre feche arquivos, conexões, etc., no finally.
  3. Evite catch vazio: Nunca deixe um bloco catch sem tratamento (isso esconde erros).
  4. Documente exceções: Use throws na assinatura do método se ele não tratar uma exceção.

Exemplo completo (leitura de arquivo com tratamento de erros)

import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class ExemploCompleto {
    public static void main(String[] args) {
        BufferedReader leitor = null;
        try {
            leitor = new BufferedReader(new FileReader("arquivo.txt"));
            String linha;
            while ((linha = leitor.readLine()) != null) {
                System.out.println(linha);
            }
        } catch (IOException e) {
            System.out.println("Erro ao ler o arquivo: " + e.getMessage());
        } finally {
            if (leitor != null) {
                try {
                    leitor.close();
                } catch (IOException e) {
                    System.out.println("Erro ao fechar o leitor: " + e.getMessage());
                }
            }
        }
    }
}
